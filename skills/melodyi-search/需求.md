# melodyi-search

提供一个工具，用于提供搜索能力。使用python语言开发。

## 需求

### 功能性需求
1. 支持多家供应商。第一版支持brave search api、firecrawl 本地部署、Tavily、exa、minimax （中国大陆）搜索接口、searxng本地部署。
2. 根据配置决定启用上述供应商中的哪些。
3. 工具的输入输出格式上，我们要先获取这些供应商的api定义，然后再进一步分析哪些参数需要统一抽取出来。  我初步的设想是query参数必填，然后需要具备 最大返回数量、时间、domains等。 如果有些api不具备时间控制，就在关键字上带一些时间的内容；如果不具备domains的控制，就硬编码在返回结果上进行过滤。  当然这个设想是初步的，根据分析完api的定义后再确认。
4. 按照多种规则进行供应商切换。例如自动负载均衡、按照固定顺序调用。并且在遇到报错、配额不足、限流等场景自动触发切换、等待、重试等机制。
5. 提供"对比模式"。这个模式用于分析每家搜索供应商的能力。开启后，每次搜索都同时触发多家供应商进行搜索，并记录。返回最快的结果给调用方，但是记录所有的结果，用于后续进行数据分析，用于对比查看供应商质量。

### 其他需求
1. 架构上基于DDD模式，定义好核心模块和输入输出结构后，也便于集成和二次开发。代码后续会被用于：直接把代码封装为tools、提供为cli工具等。第一版需要能通过命令启动，也能被安装为cli。并提供skill.md，便于agent使用。
2. 对于第一点，一个重要的标准是其中任何一个provider的实现，都能简地单独取出并二次开发、封装和使用。
3. 需要对错误，和错误需要携带提示词进行封装。这样搜索得到各类错误的时候，可以在返回中补充一些针对性提示词来引导agent后续的行为，让他进行修复或者规避等（例如需要进行什么样的配置，传入什么样的参数，或者告诉用户什么样的信息）。
4. 需要有清晰的日志记录，包括耗时信息等指标。尤其是目前处理开发阶段，我需要能仅通过日志就快速定位代码的bug或问题。
5. skill.md的描述、介绍的提示词中，不透露provider信息。

## 提示词参考

### Claude Code

完整提示词：
```markdown
- Allows Claude to search the web and use the results to inform responses
- Provides up-to-date information for current events and recent data
- Returns search result information formatted as search result blocks, including links as markdown hyperlinks
- Use this tool for accessing information beyond Claude's knowledge cutoff
- Searches are performed automatically within a single API call

CRITICAL REQUIREMENT - You MUST follow this:
- After answering the user's question, you MUST include a "Sources:" section at the end of your response
- In the Sources section, list all relevant URLs from the search results as markdown hyperlinks: [Title](URL)
- This is MANDATORY - never skip including sources in your response
- Example format:

[Your answer here]
Sources:
  - [Source Title 1](https://example.com/1)
  - [Source Title 2](https://example.com/2)

Usage notes:
- Domain filtering is supported to include or block specific websites
- Web search is only available in the US

IMPORTANT - Use the correct year in search queries:
- The current month is ${currentMonthYear}. You MUST use this year when searching for recent information, documentation, or current events.
- Example: If the user asks for "latest React docs", search for "React documentation" with the current year, NOT last year
```

重点关注：
这个提示词重点介绍了时间观念（时效性）和Agent答复用户时需要提供来源信息。我们的通用模式可能无法动态注入日期信息，可以让agent自行确认下当前年月日信息，部分agent可能默认就会注入日期，如果已有日期了就让他用已有的。
另外还需要补充对tool的出入参的描述等。
